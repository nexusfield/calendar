## Demo Flow (draft)

1. **The problem (2 min)** — institutional knowledge fragmented across email, Slack, PDFs, docs. Ken already believes this. Just confirm.
2. **What Obsidian is (3 min)** — local markdown + bidirectional links + graph view. Not a SaaS, not a database — a filesystem with smart rendering.
3. **Live tour of my AIL vault (10 min)** —
   - `people/` — one note per teammate, cross-linked
   - `domain/` — institutional finance concepts that keep coming up
   - `process/` — how we do things (coding standards, eval workflows, etc.)
   - `projects/` — active work with tasks
   - `daily/` — work logs that auto-link forward
4. **The graph (3 min)** — show the graph view; demonstrate how concepts cluster
5. **AI angle (5 min)** — Claude Code reads the vault; I can ask questions about stored knowledge; vault becomes queryable context for AI tools. This is the hook for Ken given his AI thesis.
6. **What adoption would look like (5 min)** — team-shared vault vs. individual vaults that sync; git-backed; no vendor lock-in.
7. **Q&A (5 min)**
